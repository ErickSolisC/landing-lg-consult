Preguntas teóricas:
1. Explica la diferencia entre git merge y git rebase.
    -git merce: Basicamente cambina dos ramas creando un merge
    -git rebase: Basicamente aplica los commits de una rama parametrizando el historial.

2. Explique que hace git pull y git fetch
    -git pull: Trae los cambios del repositorio y los combina en la rama local
    -git fetch: Trae los cambios del repo remoto pero este no cruza los cambios con la rama local.

3. ¿Qué significa un conflicto en Git y cómo lo resolverías?
    - Conflicto se da cuando dos ramas han modfiicado la misma linea de un archivo.
        - Para resolver esto se debe validar que cambios se aplicaran para mantener la version correcta
        - Luego se guardan los cambios y se suben.
    Adicional existen varias formas de poder resolver pero inicial mente para no crear ramas alternas esta es la solucion mas comun.

3. ¿Cuál es la diferencia entre == y === en JavaScript?
    - == en JavaScript tiene como funcion comparar valores sin importar su tipo de valor no es tan estricto.
    - === este es mas estricto porque a diferencia de == este puede validar el valor y su tipo y esto hace que sea mas robusta la validacion.

Sección 3: SQL:
1. Diferencia entre INNER JOIN y LEFT JOIN.
    - INNER JOIN: Devuel solo los registros que tiene coincidencia entre dos tablas
    - LEFT JOIN: Devuleve todas las filas de la tabla izquierda y parseando Null las de la derecha si no encuentra coincidencia

2. Explica clave primaria y clave foránea.
    - Primaria: Identificador unico de una tabla
    - Foranea: Referencia a una llave primaria en otra tabla, y tiene como objetivo establecer una relacion entre tablas.
